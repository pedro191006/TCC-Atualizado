Transações e Controle de Concorrência

6.1 Conceitos de transações

Uma transação é uma sequência de operações executadas em um banco de dados como uma unidade atômica e indivisível. Garantindo que os dados permaneçam consistentes e que não ocorram alterações incorretas no banco de dados.

6.2 Controle de concorrência para garantir a consistência dos dados

Controle de concorrência é quando, em um banco de dados, usuários distintos tentam acessar a mesma informação e então é feito um controle entre essas transações. É necessário conhecer as propriedades comumente chamadas de ACID (acrônimo de Atomicidade, Consistência, Isolamento e Durabilidade).
